Introduction

Introducing Repl.it Multiplayer. It allows you to write code and debug together with one or more partners, straight from your browser. We believe that one of the biggest advantages this brings to the table, is the ability to do real pair-programming, really well remotely.

Pair programming is the practice of pairing up two or more developers to work on the same task. The idea here is that “two heads are better than one” when solving complex tasks. It has been gaining popularity over the last few years and has been becoming easier to implement as remote coding platforms became available, such as Repl.it.

The most attractive aspect of pair programming is that two or more developers can often think of more creative solutions than a single developer would have been able to. Another great argument for pair programming is that it encourages knowledge sharing, often between more experienced deveopers and juniors. Pairs are encouraged to evaluate solutions and possible roadblocks before getting started on a problem. An added perk is that more people now have context on that piece of code incase one has to move to another team or goes on vacation. Lastly, pair programming helps immensely with in-office and remote onboarding.

The most basic requirement for a pair-programming session is that there are two or more developers working on the same piece of code, with an open stream of communication. If you already know what you’re doing you can get started right away.

If you’re new to the idea, there are many different collaborative techniques that may help make your session more productive depending on your goals. These are some of the most widely used:

  • The Ping-Pong method is usually based on Test-Driven Development style and requires that one person writes tests while the other person makes the tests pass. An important factor here is to make sure that members alternte between writing tests and passing them.
  • The Driver-Navigator method suggest one person typing (the driver) and the other person navigating. This method requires both participants to be fully engaged with the code as the driver and navigator would ideally switch roles every hour or so.
  • Unstructured pairing usually naturally occurs when two or more people colaborate on a project or a specific problem. Multiplayer is ideal for this as it allows you to spin up a project quickly within your browser, without needing to clone a project or install multiple dependencies.

There are multiple popular “video-sharing” options currently available and most of these offer screen sharing, which is great for someone doing a presentation, but not great for pair programming. Once you have spent more than a few minutes trying to describe an idea or a function to someone over screenshare, you’ll know that it’s particularly frustrating not being able to just take the wheel and show what you mean. Another downside to using video screensharing services is that it can quickly become chaotic when there are three or more people on the call.

Another option for pair programming can be done using git (eg. via GitHub or similar). While this is a great option for asynchronous, long-term projects, it can get tedious to try and communicate via pull requests and comments, especially for smaller repositories. Repl.it has the benefit that you could import git projects to your Multiplayer IDE, making collaboration on existing projects effortless. This means no push and pull, no PR’s, no confusing merge conflicts. Similar to Google Doc’s synchronised editing on the same file, all your code is live and up to date, for every member of the team.

Pair programming with Repl.it

Besides pair-programming, Multiplayer is useful for mentoring, resolving a bug or seeking help on a specific task. Many companies are becoming increasingly open to remote work, so Repl.it Multiplayer could also prove beneficial when conducting coding interviews and on-boarding with new team members.

All you need to get started is a stable internet connection and to be signed into your account. You can either start a new Multiplayer Repl or join an existing project via an invite link. To add a collaborator to your Repl, click on the “share” icon in the top right corner, which will allow you to invite collaborators via an email invite or a unique join link. This is also where you’ll find the embed feature to add your code snippet to your personal blog, StackOverflow, porfolio etc.

To start a chat, simply click on the purple chat box icon in the bottom right corner and start typing. Multiplayer is session-based so when the admin person leaves, you will be able to continue working on your project. However, if you close your current Multiplayer tab, you will not be able to use the same invite link to join the session. Simply ask a collaborator to add you again. We are currently working on a feature that will allow admin users to remove collaborators, and may form part of the Teams Plan.

At the moment, Multiplayer can only be used if you’re signed in to Repl.it. We are actively working on an anonymous Multiplayer functionality, but in the meantime, you can send collaborators a unique join link to your project, after which they will be redirected to sign in using Google, GitHub or Facebook. Stay updated on our progress here.
